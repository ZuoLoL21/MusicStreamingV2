# API Service Event Ingestion

> **⚠️ INTERNAL SERVICE ONLY**
> This service is NOT directly accessible to clients. All client requests must go through gateway_api (port 8080).
> This documentation is for backend developers and service-to-service communication reference only.

This document describes the RESTful API for the Event Ingestion microservice, which handles real-time analytics events and stores them in ClickHouse for data warehouse and ML pipelines.

## Access

**Direct URL:** `http://localhost:8005` (internal only)
**Accessed via gateway_api:** `http://localhost:8080/events/*`

## Authentication

All endpoints except `/health` require Service JWT tokens (automatically added by gateway_api).

### Authentication Header

```http
Authorization: Bearer <service_jwt_token>
```

**Note:** Service JWTs are automatically generated by gateway_api when proxying client requests. Clients should never generate Service JWTs directly.

## Event Endpoints

### Ingest Listen Event

Record a music listen event with user engagement metrics.

**Endpoint:** `POST /events/listen`

**Client Access:** `POST http://localhost:8080/events/listen` (requires Normal JWT)

**Authentication:** Service JWT required (auto-added by gateway_api)

**Request Body:**
```json
{
  "user_uuid": "123e4567-e89b-12d3-a456-426614174000",
  "music_uuid": "223e4567-e89b-12d3-a456-426614174000",
  "artist_uuid": "323e4567-e89b-12d3-a456-426614174000",
  "album_uuid": "423e4567-e89b-12d3-a456-426614174000",
  "listen_duration_seconds": 120,
  "track_duration_seconds": 180,
  "completion_ratio": 0.67
}
```

**Request Fields:**
| Field | Type | Required | Description | Validation |
|-------|------|----------|-------------|------------|
| `user_uuid` | UUID | Yes | User who listened to the track | Must be valid UUID (not nil) |
| `music_uuid` | UUID | Yes | Music track UUID | Must be valid UUID (not nil) |
| `artist_uuid` | UUID | Yes | Artist UUID | Must be valid UUID (not nil) |
| `album_uuid` | UUID | No | Album UUID (if track is in an album) | Must be valid UUID if provided |
| `listen_duration_seconds` | integer | No | How long the user listened (seconds) | Must be >= 0 if provided (default: 0) |
| `track_duration_seconds` | integer | No | Total track duration (seconds) | **No server-side validation** - accepts any integer (default: 0) |
| `completion_ratio` | float | Yes | Percentage of track completed (0-1) | Must be between 0.0 and 1.0 |

**Note:** `track_duration_seconds` has no validation in the current implementation. Clients should ensure they send accurate positive values representing the actual track duration.

**Response:**
```json
{
  "success": true
}
```

**Status Codes:**
- `200 OK` - Event ingested successfully
- `400 Bad Request` - Invalid request body or validation error
- `401 Unauthorized` - Missing or invalid Service JWT
- `500 Internal Server Error` - ClickHouse insertion failed

### Ingest Theme Event

Assign or update a music track's theme/genre classification.

**Endpoint:** `POST /events/theme`

**Client Access:** `POST http://localhost:8080/events/theme` (requires Normal JWT)

**Authentication:** Service JWT required (auto-added by gateway_api)

**Request Body:**
```json
{
  "music_uuid": "223e4567-e89b-12d3-a456-426614174000",
  "theme": "rock"
}
```

**Request Fields:**
| Field | Type | Required | Description | Validation |
|-------|------|----------|-------------|------------|
| `music_uuid` | UUID | Yes | Music track UUID | Must be valid UUID |
| `theme` | string | Yes | Theme/genre name | Cannot be empty or whitespace-only |

**Response:**
```json
{
  "success": true
}
```

**Notes:**
- This endpoint performs an **upsert** operation - if the theme for a music track already exists, it will be updated
- Theme names are case-sensitive

**Status Codes:**
- `200 OK` - Theme upserted successfully
- `400 Bad Request` - Missing music_uuid/theme or validation error
- `401 Unauthorized` - Missing or invalid Service JWT
- `500 Internal Server Error` - ClickHouse upsert failed

### Ingest User Dimension Event

Create or update user dimension data for analytics.

**Endpoint:** `POST /events/user`

**Client Access:** `POST http://localhost:8080/events/user` (requires Normal JWT)

**Authentication:** Service JWT required (auto-added by gateway_api)

**Request Body:**
```json
{
  "user_uuid": "123e4567-e89b-12d3-a456-426614174000",
  "created_at": "2024-01-01T00:00:00Z",
  "country": "US"
}
```

**Request Fields:**
| Field | Type | Required | Description | Validation |
|-------|------|----------|-------------|------------|
| `user_uuid` | UUID | Yes | User's unique identifier | Must be valid UUID |
| `created_at` | timestamp | Yes | User account creation timestamp | ISO 8601 format |
| `country` | string | Yes | User's country code | Must be uppercase ISO 3166-1 alpha-2 (2 letters, e.g., "US", "CA", "GB") |

**Country Code Validation:**
- Must be exactly 2 characters
- Must be uppercase letters only (A-Z)
- Must be valid ISO 3166-1 alpha-2 code
- Examples: `"US"`, `"CA"`, `"GB"`, `"FR"`, `"DE"`, `"JP"`

**Response:**
```json
{
  "success": true
}
```

**Notes:**
- This endpoint performs an **upsert** operation - if the user dimension already exists, it will be updated

**Status Codes:**
- `200 OK` - User dimension upserted successfully
- `400 Bad Request` - Missing required fields or invalid country code format
- `401 Unauthorized` - Missing or invalid Service JWT
- `500 Internal Server Error` - ClickHouse upsert failed

**Validation Error Examples:**
```json
{
  "error": "Country must be uppercase ISO 3166-1 alpha-2 code (letters only)"
}
```

```json
{
  "error": "Country must be ISO 3166-1 alpha-2 code (2 chars)"
}
```

```json
{
  "error": "Failed to ingest event"
}
```
Status: `500 Internal Server Error`

- User behavior analysis
- Theme-based recommendations (service_bandit_system)

- Geographic distribution analysis
